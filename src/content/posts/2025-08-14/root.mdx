一直以来都想从pixelos换回coloros。尝试了很多方法，从刷第三方recovery---twrp，到adb sideload，都不行。绝望之下，去咸鱼花了钱买服务。没想到他用的是最基础的

## PiexlOS

### 优点

- 精简
- 有免费的无限相册保存空间，由谷歌相册提供，据说不限量。适合保存小电影（不是）
- 很流畅。60hz看上去比coloros的要好（也可能是动画缩放太小了）
- 原生支持gemini
- 有esim功能，（需要插入入9esim的卡等），这样就可以在系统中更换了，esim支持度比coloros好
- 有极暗模式,非常暗,很适合晚上看手机(´-ω-`) zZZ
- 手电筒亮度1-100随便调，coloros只有4个档位
- 没有所谓的监控和云控
- 不会自己切换WiFi和数据上网

### 缺点
- 没有小窗模式，只有选择性的分屏（摊手 ＝＿＝）
- 没有很多附属的功能，这也算高度精简吧。比如识别文字
- 高度依赖谷歌服务
- 不知道国内应用的通知支持程度，（我发现钉钉的离线功能不支持，我也不是很了解通知推送的渠道）
- 影像与硬件调校弱：缺少厂商级相机算法与色彩调校，夜景/HDR/视频防抖体验不如 ColorOS。**前摄聚焦能力有点差**
- 本地化功能不足：骚扰拦截、通话录音、隐私空间、文件管理增强、系统级跨端互联（如小布/OPPO 生态）等缺失或替代方案不佳。
**特别是默认的短信和电话app更是一坨**，但是可以自己更换替代品，对我来说还是国产的这些更符合我的审美
- 字体大小千奇百怪的，尤其是微信，怎么看都不顺眼
- 生态不好，很多教程找不到（狗头），比如工具箱、内核？
- 人脸解锁不太好,有时候很难解锁
- 原生的皮肤太丑了,尤其是系统,看多了非常嫌弃
- 天气地图不好搞,默认的app和商店里的同类app也是一坨.不知道ip会不会送中

## ColorOS

### 优点
- 稳定与省心：系统级细节（通知到达率等）更稳，长时间使用更省心(稳！(๑•̀ㅂ•́)و✧)。
- 影像体验：相机算法、HDR、夜景/视频防抖、色彩调校更成熟，直出更讨喜。
- 本地化功能：小窗/跨端互联/骚扰拦截/通话录音/隐私空间/文件管理增强等一应俱全。
- 生态适配：国内 App 通知、深色模式/字体/权限适配更好，少踩坑。
- 售后与 OTA：官方固件与增量 OTA 更顺滑，遇到问题有回退路径。

### 缺点
- 预装与推荐内容较多，部分系统/内置应用存在信息流与广告开关，需逐项关闭。（当然root之后，安装各种模块可以大幅度的改善体验）
- 系统自由度较低，权限/默认应用/桌面自定义可玩性不如 AOSP/第三方 ROM。
- 生态绑定深（云服务/相册/互联），迁移到其它平台成本较高。

# fastboot刷机

## 注意事项
1.出现官方的recovery(让你选择语言时),不要选(千万别点 ；´Д`)
- 备份所有数据（通讯录/短信/相册/聊天记录/银行卡绑定App），解锁或刷机可能清空数据。
- 保证电量≥60%，使用原装或优质数据线，尽量直连主机板 USB 口（避开集线器）。
- 仅刷与机型/地区匹配的官方包，避免跨地区/跨运营商固件（易基带/频段异常）。
- 使用最新 Platform-Tools（https://developer.android.com/studio/releases/platform-tools），避免旧版 fastboot 兼容问题。
- Windows 需安装设备驱动（ADB/Fastboot/Qualcomm），必要时禁用驱动签名强制安装。
- 不要误动 EFS/Modem/ persist 等分区；这些分区与基带/IMEI/传感器密切相关。
- 解锁 BL 会清空数据；若计划 Root，请勿在刷完后立刻“上锁”，否则可能卡砖。
- 若开机进入官方 Recovery 出现“选择语言”界面，请勿继续初始化，优先重启回 Bootloader 再处理。
- 动态分区设备需进入 fastbootd 才能刷 super.img 或使用 fastboot update。(我也不知道分区的知识)


## 所需文件
- 平台工具：platform-tools（解压到英文路径，如 C:\adb）。
- 对应机型的完整官方包（ColorOS/PixelOS 线刷包，含 payload.bin 或官方脚本）。
-（可选）payload-dumper-go 用于解包 payload.bin（https://github.com/ssut/payload-dumper-go）。

## 进入 Fastboot / Fastbootd
```
adb reboot bootloader              # 重启到 Fastboot（Bootloader）
fastboot devices                   # 确认设备已连接
fastboot reboot fastboot           # 进入 Fastbootd（用户态），用于刷 super 等动态分区
```
如无法进入 fastbootd，可在 Fastboot 界面用音量键选择，或升级 platform-tools 后再试。

## 主流 Root 方案对比分析,**由gemini总结的**

| 方案 | 优点 | 缺点 | 适合人群 |
| :--- | :--- | :--- | :--- |
| **[1] Magisk** | 👑 **最稳定、最主流**<br />模块生态最丰富，几乎所有教程都围绕它<br />安装和使用最简单省心 | 目标最大，容易被各种App检测针对 | **绝大多数用户，特别是新手**、追求稳定、不想折腾的小可爱。 |
| **[2] Kitsune (Magisk Delta)** | 🎭 **隐藏能力更强**<br />继承Magisk大部分功能<br />内置多种隐藏手段，对付银行等App检测效果好 | 相比原版Magisk，稳定性可能稍逊一筹 | 需要Root但又想尽量躲过App检测，特别是**金融、游戏类App**的用户。 |
| **[3] Alpha** | 🧪 **最新、最激进**<br />集成了各种最新的Root技术和隐藏方案<br />功能非常强大，更新快 | **极不稳定**，可能会导致各种奇奇怪怪的问题，甚至手机变砖 | 喜欢尝鲜、不怕折腾、有一定救砖能力的**搞机大神**。 |
| **[4] APatch** | 🧩 **内核级修补**<br />直接修补内核，某些方面隐藏性更好<br />与Magisk模块部分兼容 | **操作门槛高**，需要理解内核知识<br />如果内核不支持，就无法使用 | 动手能力强，对内核有一定了解的**高级玩家**。 |
| **[5] KernelSU** | 🚀 **新世代方案**<br />从内核层面实现Root，隐藏性极佳<br />对系统影响小，理论上更稳定、更省电 | **需要内核源码支持**，并非所有机型都能用<br />模块生态正在发展中，不如Magisk丰富 | **所持机型官方支持KernelSU**，追求极致隐藏和性能的用户。 |
| **[6] KernelSU-Next** | 🌟 **KernelSU的未来**<br />KernelSU的测试版或开发者预览版<br />包含最新的功能和修复 | **不稳定**，仅供测试 | 想要提前体验KernelSU新功能的**开发者或高级玩家**。 |
| **[7] SukiSU-Ultra** | 💖 **未知的新秀**<br />这似乎是一个非常小众或新兴的方案，公开信息极少 | **生态、稳定性、安全性未知**，可能是个人开发者作品 | **极度爱折腾，不畏风险的探索者**。不推荐普通宝宝尝试！ |
